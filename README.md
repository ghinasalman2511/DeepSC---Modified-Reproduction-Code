# DeepSC---Modified-Reproduction-Code
This project is a modified version of the original code available at https://github.com/13274086/DeepSC/tree/master, created to reproduce some results from the original article: https://arxiv.org/abs/2006.10685.
--------
This code is a modified version of the original implementation available at https://github.com/13274086/DeepSC/tree/master, which corresponds to the paper "Deep Learning Enabled Semantic Communication Systems" by Huiqiang Xie et al. (arXiv link: https://arxiv.org/abs/2006.10685). 
In this modified code, we reproduce the key results presented in the original paper, specifically Figure 6(a) and Figure 6(b), by training the DeepSC model on the full European Parliament dataset (2 million sentences) for 80 epochs with the same architecture parameters (3 Transformer layers, d_model=128, 8 attention heads, batch size 128). 
We modified the original code by adjusting the number of layers from 4 to 3 as specified in Table I of the paper, changing the hardcoded dataset paths to local paths (./data/), and extending the performance evaluation script to compute BLEU scores for 1-gram, 2-gram, 3-gram, and 4-gram simultaneously rather than only 1-gram. Additionally, we added automatic plotting functionality using matplotlib to generate training loss curves, mutual information evolution plots, and BLEU score versus SNR figures directly from the evaluation results. 
The training was conducted using PyTorch 2.7.1+cu118 with CUDA acceleration on an NVIDIA RTX 3050 6GB GPU, and we successfully maintained the original batch size of 128 without any reduction. Data preprocessing is performed by running preprocess_text.py, which filters sentences with lengths between 4 and 30 words, builds a vocabulary of approximately 24,000 tokens, and splits the data into 90% training and 10% testing sets saved as pickle files in the ./data/europarl directory.
 In the directory ./data/europarl, you must save the last checkpoints for each experimental scenario (checkpoints1, checkpoints2, etc.). We also assume that the downloaded European Parliament dataset is unzipped inside the ./data/europarl directory, where the txt directory from the downloaded dataset must exist within ./data/europarl. 
Our experimental results, including training convergence analysis and BLEU performance measurements, are presented and compared with the original paper's reported values.
